**ERP Διαχείρισης Παραγγελιών, Αποθήκης και Δημιουργίας Στατιστικών Καταστήματος B2B**

- **Βασικές Λειτουργίες:**

**Login:** Ο χρήστης θα μπορεί να κάνει login σαν ταμίας (διαχείριση παραγγελιών), αποθηκάριος, οικονομικό τμήμα ή admin-manager (Πρόσβαση σε όλες τις λειτουργίες).

**Διαχείριση παραγγελιών:**
  - Ο ταμίας θα μπορεί να καταγράφει μία παραγγελία για λογαριασμό του πελάτη. Τα προϊόντα θα περιέχονται σε έναν κατάλογο, το καθένα με μοναδικό id, όπου θα γίνεται η επιλογή της ποσότητας του κάθε προϊόντος, θα υπολογίζεται το κόστος των προϊόντων, ο ΦΠΑ και το τελικό κόστος, και θα δημιουργείται μία αναφορά παραγγελίας, η οποία θα αποθηκεύεται στο ιστορικό.
  - Ο διαχειριστής παραγγελιών θα μπορεί να επεξεργάζεται νέους πελάτες (δημιουργία νέων πελατών, διαγραφή, επεξεργασία στοιχείων).
  - Θα δίνεται δυνατότητα προβολής πληροφοριών πελάτη (στοιχεία, ιστορικό κ.α.)
  - Κατά την παραγγελία, θα γίνεται ερώτηση για τον ποιον πελάτη αφορά η παραγγελία, με τρία πιθανά αποτελέσματα (1. Ήδη υπάρχων πελάτης, θα δίνεται το id, 2. Νέος πελάτης: Θα συμπληρώνονται τα στοιχεία του και κατόπιν, θα πραγματοποιείται η παραγγελία, 3. Επισκέπτης Πελάτης: Δεν θα αναγράφονται τα στοιχεία πελάτη, δεν θα έχει δυνατότητα επιβραβεύσεων και δεν θα διατηρείται ιστορικό στο όνομά του)
  - CustomerLoyaltySystem: Θα δίνεται η δυνατότητα στην πελάτη μέσω της ταυτοποίησης του να μαζεύει πόντους από τις παραγγελίες του και να μπορεί να απολαμβάνει διάφορα οφέλη από τη συγκέντρωση πόντων.
  
**Παρακολούθηση αποθήκης:**
  - Πολλά προϊόντα που παραγγέλνονται, θα βρίσκονται στην αποθήκη του καταστήματος . Το σύστημα θα πρέπει να παρακολουθεί το ρυθμό με τον οποίο οι παραγγελίες θα μειώνουν τα αποθέματα της αποθήκης και να προχωρά στην παραγγελία επιπρόσθετων προϊόντων μέσω του χρήστη.
  - Τα προϊόντα της αποθήκης, μπορεί να είναι οργανωμένα σε κατηγορίες (π.χ. για κατάστημα ηλεκτρονικών ειδών: υπολογιστές, κινητά, εκτυπωτές, τηλεοράσεις, λευκές συσκευές κ.α.) και θα επιτρέπεται η αναζήτηση είτε στο σύνολό τους, είτε ανά κατηγορία (πχ αναζήτηση μόνο στα κινητά ή αναζήτηση σε όλα τα προϊόντα) (αυτό σημαίνει πως στην κλάση προϊόν, μία μεταβλητή στιγμιοτύπου, θα είναι ο τύπος προϊόντος).
  -  Θα δίνεται δυνατότητα επεξεργασίας της λίστας προϊόντων (δημιουργία νέου, επεξεργασία, διαγραφή).
  -  Θα δίνεται η δυνατότητα προβολής των πληροφοριών των προϊόντων (id, αρ. αποθέματος, τιμή πώλησης κ.α.)
  - Τα στοιχεία της παραγγελίας αποθεμάτων (όνομα εταιρίας που προμηθεύτηκε τα αποθέματα, προϊόντα και ποσότητες που προμηθεύτηκε, κόστος παραγγελίας κ.λπ.) θα συμπληρώνονται χειροκίνητα από το χρήστη. Θα διατηρείται και ιστορικό προμηθειών.
  - Κατά την παραγγελία, θα αναγράφεται η κατάσταση του αποθέματος στην οθόνη παραγγελίας («σε απόθεμα», «τελευταία κομμάτια», «εξαντλημένο»).

***Τα στοιχεία με αστερίσκο θα μπορούν να γίνονται και από το μενού παραγγελίας.**

**Οικονομικό Τμήμα - HR** : Θα γίνεται αξιοποίηση των στοιχείων των ιστορικών για τη δημιουργία στατιστικών (έσοδα, έξοδα, προϊόντα με μεγαλύτερη-μικρότερη ζήτηση, κ.α.) και για την πρόταση διαφόρων ενεργειών π.χ:
    - κάποιο προϊόν που δεν έχει ζήτηση, ή έχει χαμηλές αξιολογήσεις, να προτείνεται να αφαιρεθεί ή να γίνει έκπτωση κ.α.
    - να εμφανίζονται οι πωλήσεις ανά ημέρα, οι μέσες ημερήσιες πωλήσεις, τα ημερήσια έσοδα παραγγελιών και έξοδα ανεφοδιασμού της αποθήκης κ.α.
    - να εμφανίζονται οι συνολικές αγορές ενός πελάτη, οι μέση αξία παραγγελίας ανά πελάτη, η συχνότητα αγοράς κ.α.
    - κατάταξη των προϊόντων με βάση τη ζήτησή τους και διάφορα άλλα κριτήρια.
  - Θα μπορούν να δημιουργηθούν και διαγράμματα σε αυτό το κομμάτι.
  - Θα προτείνονται διάφορες ενέργειες με βάση τις παραπάνω αναλύσεις (π.χ. προωθητικές ενέργειες, έκπτωση κ.λπ.)
  - Διαχείριση Προσωπικού:  Να δίνεται η δυνατότητα πρόσληψης ή απόλυσης προσωπικού, ο κάθε εργαζόμενος έχει μισθοδοσία, η  οποία επηρεάζει τα έξοδα της επιχείρησης και συνεπώς τα συνολικά κέρδη.
